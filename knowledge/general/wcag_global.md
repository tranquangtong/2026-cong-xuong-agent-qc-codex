# WCAG Global Baseline

This file captures shared accessibility guidance for all specialist agents.

## WCAG 2.2 Baseline
- Use WCAG 2.2 as the default accessibility benchmark for web and e-learning QC.
- Prefer WCAG 2.2 guidance over older WCAG 2.0 or 2.1-only interpretations when a newer requirement exists.
- Treat accessibility findings as cross-cutting issues relevant to content, design, interaction, and assessments.

## Visual Accessibility Rules
- Normal-size text should meet at least 4.5:1 contrast.
- Large text should meet at least 3:1 contrast.
- Non-text UI components and meaningful graphical objects should meet at least 3:1 contrast where applicable.
- Do not rely on color alone to communicate meaning, status, or error.
- Focus indicators should be clearly visible and have sufficient contrast.
- Touch targets and interactive targets should be large enough and spaced clearly to avoid accidental activation.

## Layout And Interaction Rules
- Preserve readable hierarchy with clear headings, labels, grouping, and navigation cues.
- Keep spacing and alignment consistent so users can scan interfaces predictably.
- Avoid hidden content states that are difficult to discover or operate.
- Ensure interactive controls are identifiable, consistently styled, and understandable in context.
- Feedback for correct or incorrect answers must appear only after a valid learner action.

## Text And Content Rules
- Use plain, unambiguous instructional wording where possible.
- Maintain consistent terminology, capitalization, and language variant within the same learning item.
- Ensure labels, instructions, and helper text remain readable when zoomed or viewed on smaller screens.

## Source Note
- Based on W3C WCAG 2.2 as the current global reference standard.
