# Content QC Review Rubric

## Supported Source Types

This workflow is optimized for:

- `pdf`
- `csv`
- `docx`
- subtitles or copied text
- pre-resolved Figma text
- screenshots only when visible text is legible enough to inspect

## Review Focus

Check for:

- grammar and spelling
- British English consistency
- terminology alignment
- learner-facing clarity
- subtitle or storyboard consistency
- CTA and label consistency when visible in the source

## Figma Handling

Do not claim deep content coverage from a raw Figma link alone.

- If the frame text has been resolved into text or a readable image, review it normally
- If the link is unresolved, raise a limitation finding or summary note
- If both content and visual review are needed, keep text findings separate from layout findings

## Finding Quality Bar

Strong content findings should:

- cite source cues such as page, paragraph, row, or frame when available
- distinguish confirmed text defects from terminology or style suggestions
- recommend the corrected phrasing or the change direction clearly

## Style Expectations

Default project expectations:

- British English
- consistent learner-facing terminology
- concise and readable e-learning copy
- no overclaiming when only partial text is available
