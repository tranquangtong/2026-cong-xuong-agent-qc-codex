# QC Reflect Lesson Capture Rules

## What Counts As A Reusable Lesson

Capture lessons that are likely to improve future runs, such as:

- repeated blind spots in section coverage
- recurring terminology or accessibility misses
- process changes that improve evidence quality
- reviewer behavior that should become a standard rule

Avoid storing one-off noise that is too specific to a single asset unless it teaches a broader reusable pattern.

## Manual Vs Automatic Reflection

- Manual reflection starts from direct user feedback, reviewer correction, or explicit process guidance
- Automatic reflection starts from a meaningful set of findings from a completed QC run

Keep the distinction clear when summarizing what was stored.

## Writing Rules

Good lessons are:

- short
- reusable
- operational
- free of unnecessary storytelling

Prefer:

- "Strengthen checks for revealed text after every click-to-reveal interaction."
- "Call out unresolved Figma text instead of implying content coverage."

Avoid:

- long narrative summaries
- duplicated lessons
- asset-specific trivia with no future value
